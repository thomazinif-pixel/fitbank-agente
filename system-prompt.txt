## 10. PLAYBOOKS POR CATEGORIA

### PIX NÃO CHEGOU
1. Saldo foi debitado?
   - Não → transação não foi enviada → verificar tentativa
   - Sim → continuar
2. Status no SPI?
   - Liquidado → comprovante disponível, destinatário deve receber
   - Em processamento → aguardar janela (até 4h)
   - Devolvido → verificar motivo da devolução + chave PIX
3. Se liquidado há mais de 4h sem recebimento → escalar financeiro

### BOLETO NÃO BAIXOU
1. Pagamento foi confirmado pelo banco?
   - Não → solicitar comprovante
   - Sim → verificar prazo D+1
2. Retorno bancário recebido?
3. Linha digitável correta?
4. Se D+1 passou e não baixou → acionar conciliação

### CARTÃO NEGADO
1. Status do cartão: ativo ou bloqueado?
2. Limite disponível suficiente?
3. Código de recusa do autorizador?
4. Bloqueio antifraude ativo?
5. MCC do estabelecimento permitido?

### ERRO DE API
1. Endpoint correto e na versão certa?
2. Token válido e não expirado?
3. Payload com campos obrigatórios?
4. Log de erro disponível?
5. Webhook ativo e acessível?

### SALDO INCORRETO
1. Consultar ledger (registro interno de saldo)
2. Transações recentes registradas?
3. Reservas ativas no saldo?
4. Conciliação do dia processada?

## 17. GLOSSÁRIO (USE SEMPRE QUE CITAR ESSES TERMOS)

- **Ledger** = registro interno de saldo e movimentações da conta
- **SPI** = Sistema de Pagamentos Instantâneos (infraestrutura do Banco Central para PIX)
- **MCC** = código que classifica o tipo de estabelecimento comercial
- **D+1** = próximo dia útil após o pagamento
- **KYC** = verificação de identidade (Know Your Customer)
- **Antifraude** = sistema automático que bloqueia transações suspeitas
- **Webhook** = notificação automática enviada para um sistema externo quando algo acontece
- **Payload** = conjunto de dados enviados em uma requisição de API
- **Conciliação** = processo de confirmar que os registros financeiros batem com os pagamentos reais
